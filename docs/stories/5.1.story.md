## Story

**As a** administrador cadastrando ou editando um empreendimento,
**I want** poder registrar a área total, os dados jurídicos da SPE, inscrições estadual/municipal, e os registros de incorporação e alvarás diretamente no formulário,
**so that** o cadastro do empreendimento fique completo com as informações legais e técnicas necessárias para operações formais — sem precisar de workarounds externos.

## Acceptance Criteria

1. O campo `total_area` (string opcional) é adicionado ao card **"Informações Básicas"** do formulário, após o campo de descrição. Label: **"Área Total"**, placeholder: `"Ex: 12.500 m²"`.

2. Um novo card **"Pessoa Jurídica (SPE)"** é adicionado ao formulário, com os campos:
   - `cnpj` — input com máscara `XX.XXX.XXX/XXXX-XX` (componente `CNPJInput` de `src/components/ui/document-input.tsx`), label "CNPJ", placeholder "00.000.000/0000-00"
   - `legal_name` — input texto, label "Razão Social", maxLength 200
   - `trade_name` — input texto, label "Nome Fantasia", maxLength 200
   - `<FormDescription>` sob `cnpj` e `legal_name`: *"Quando vazio, herda os dados da organização"*

3. A máscara e validação de CNPJ são **idênticas** ao módulo `customers` PJ:
   - Máscara: `maskCNPJ` de `@cacenot/construct-pro-api-client` via `CNPJInput`
   - Validação Zod: `validateCNPJ` de `@cacenot/construct-pro-api-client` com `.refine()` — apenas quando o campo não está vazio (campo é opcional)

4. Um novo card **"Registros e Alvarás"** é adicionado, com os campos:
   - `incorporation_registry_number` — label "Registro de Incorporação (R.I.)"
   - `mother_property_registration` — label "Matrícula Mãe"
   - `cno` — label "CNO (Cadastro Nacional de Obras)", validação: exatamente 12 dígitos quando preenchido
   - `construction_permit_number` — label "Alvará de Construção"
   - `occupancy_permit_number` — label "Habite-se"

5. Um novo card **"Inscrições"** é adicionado, com os campos:
   - `state_registration` — label "Inscrição Estadual (IE)"
   - `municipal_registration` — label "Inscrição Municipal (IM)"

6. Todos os campos novos são **opcionais**: campos vazios não bloqueiam o submit do formulário.

7. Conflito de CNPJ único por tenant (409 da API) é tratado pelo `handleApiError` padrão nas pages de criação e edição — nenhuma lógica especial necessária além do que já existe.

8. Os campos novos aparecem tanto no formulário de **criação** (`pages/empreendimentos/novo/+Page.tsx`) quanto no de **edição** (`pages/empreendimentos/@id/editar/+Page.tsx`) — sem mudança de rota ou lógica de submit nessas pages.

9. Empreendimentos já cadastrados **sem** os novos campos continuam editáveis normalmente (todos os novos campos têm `defaultValue` como string vazia ou `null`).

10. `npm run build` limpo e `npm run lint` sem warnings após a implementação.

11. Sem regressão nos campos existentes: `name`, `status`, `description`, `address`, `number`, `district`, `city`, `state`, `postal_code`, `floors`, `delivery_date`, `features`.

### Validação Zod CNPJ — Padrão do Projeto

[Source: `src/schemas/customer.schema.ts` linhas 39–46]

```ts
import { validateCNPJ } from '@cacenot/construct-pro-api-client'

const projectCnpjSchema = z
  .string()
  .optional()
  .nullable()
  .refine(
    (val) => {
      if (!val) return true  // campo opcional — vazio é válido
      const digits = val.replace(/\D/g, '')
      if (digits.length !== 14) return false
      return validateCNPJ(digits)
    },
    { message: 'CNPJ inválido' }
  )
```

> **Diferença do padrão customer:** aqui o campo é opcional — adicionar guard `if (!val) return true` antes da validação.

### Validação CNO — 12 Dígitos

```ts
const projectCnoSchema = z
  .string()
  .optional()
  .nullable()
  .refine(
    (val) => !val || val.replace(/\D/g, '').length === 12,
    { message: 'CNO deve ter exatamente 12 dígitos' }
  )
```

CNO é numérico — considerar `inputMode="numeric"` e `maxLength={12}` no `<Input>`.
