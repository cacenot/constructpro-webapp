## Story

**As a** corretor usando o formulário de nova proposta,
**I want** selecionar primeiro o empreendimento e depois a unidade disponível filtrada por esse empreendimento em dois passos visuais explícitos,
**so that** eu não veja unidades de outros empreendimentos e o fluxo de preenchimento seja mais claro e guiado.

## Acceptance Criteria

1. O Step 1 do wizard exibe `<ProjectAutocomplete>` para escolha do empreendimento e `<UnitAutocomplete>` que permanece **desabilitado** até que um empreendimento seja selecionado.
2. Ao selecionar empreendimento, `<UnitAutocomplete>` é habilitado e lista **somente** unidades com `status=available` e `project_id` igual ao empreendimento selecionado.
3. Ao trocar o empreendimento (selecionar outro ou limpar seleção), o campo de unidade é **resetado** (valor limpo) e desabilitado até nova seleção.
4. O botão "Próximo" (avançar para Step 2) só está habilitado quando **ambos** empreendimento e unidade estiverem selecionados.
5. O Step 2 contém todos os campos financeiros existentes (parcelas, comissão, cliente, índice, broker) sem alteração de comportamento.
6. O formulário de **edição** (`sale-edit-form.tsx`) continua funcionando sem regressão — o wizard se aplica apenas à criação.
7. O `<UnitAutocomplete>` com `projectId` **não** faz fetch de mapa de nomes de projetos separado, pois o nome do empreendimento já é conhecido.

### Wizard State em `sale-form.tsx`

O componente `SaleForm` precisa de:

```ts
const [step, setStep] = React.useState<1 | 2>(1)
const [selectedProject, setSelectedProject] = React.useState<SelectedProject | null>(null)
```

O `selectedProject.id` é passado como `projectId` para `<UnitAutocomplete>`. O `unit_id` já é controlado pelo `react-hook-form` via `form.setValue`.

### Reset ao Trocar Empreendimento (AC: 3)

Quando `ProjectAutocomplete.onChange` disparar com `project !== selectedProject`:

```ts
setSelectedProject(project)
form.setValue('unit_id', null)  // ou o campo correto no schema
setSelectedUnit(null)           // estado local existente em sale-form.tsx:64
```

### Compatibilidade Retroativa de `UnitAutocomplete`

`projectId` deve ser **opcional**. Quando ausente, comportamento atual é preservado:
- `GET /api/v1/units` sem `project_id` (busca global)
- queryKey permanece `['units-search', debouncedSearch, undefined]` — sem invalidação extra
