## História

**Como** usuário autenticado (vendedor ou gerente),
**Quero** visualizar a listagem paginada de vendas com filtros por status,
**Para que** possa acompanhar o funil comercial e localizar rapidamente propostas em diferentes estágios.

## Critérios de Aceitação

1. Hook `useSales({ page, page_size, status })` é chamado com os parâmetros corretos e retorna dados paginados do servidor.
2. Skeleton rows são exibidos durante o carregamento inicial e após mudança de filtros/página.
3. Empty state com mensagem em português é exibido quando a API retorna lista vazia.
4. Filtro de status permite selecionar `offer`, `reserved`, `closed`, `lost` (select ou multi-select); ao limpar, exibe todos.
5. Coluna de status exibe badge colorido por `Sale.status` (cor distinta por status).
6. Paginação server-side: botões de página, info "X–Y de Z vendas", mínimo de 10 itens por página.
7. Filtros persistem durante navegação (state gerenciado no hook, não na URL).
8. Colunas responsivas: em mobile (`< md`) ocultam colunas secundárias (ex: vendedor, data criação).
9. Todos os botões de ícone possuem `<Tooltip>` com label em português.
10. Testes E2E passam 100%: `npx playwright test e2e/sales/lista-vendas.spec.ts`.

## Notas de Desenvolvimento

### Stack e Padrões Obrigatórios

[Source: CLAUDE.md]

**Stack:** React 19 + TypeScript 5.9 (strict) + Vite 7 + Tailwind CSS 4 + shadcn/ui (New York style)

**Routing:** Vike SPA mode. Rota nova: `pages/sales/+Page.tsx` → URL `/sales`. Sem SSR.

**State:** TanStack Query v5 para server state (staleTime 5min, gcTime 30min). Zustand apenas para UI state (sidebar, theme). Filtros e paginação ficam no hook `use-sales-table.ts`.

**API Client:**
```ts
import { useSales, useApiClient } from '@cacenot/construct-pro-api-client'

// Parâmetros aceitos:
const { data } = useSales({ page: 1, page_size: 10, status: ['offer', 'reserved'] })
// data.items: Sale[]
// data.total: number

// Query key para invalidação: ['sales']
```

**Table Pattern obrigatório** (CLAUDE.md):
- Hook: `src/hooks/use-sales-table.ts` — retorna `{ data, isLoading, total, hasActiveFilters, handleClearFilters, filters, pagination }`
- Columns: `src/components/sales/sales-columns.tsx` — `ColumnDef<Sale>[]`
- Table: `src/components/sales/sales-table.tsx` — `useReactTable` + `manualPagination: true`
- Pagination: `src/components/sales/sales-pagination.tsx` — shadcn Pagination + Button
- Filters: `src/components/sales/sales-filters.tsx` — recebe state/setters como props
- Page: `pages/sales/+Page.tsx` — thin orchestrator apenas

**Page size mínimo:** 10 itens por página.

**Paginação:** Server-side. `getCoreRowModel` + `manualPagination: true` + `rowCount: total`.
