# UX Spec — Story 4.6: Wizard 3 Steps com Resumo Financeiro Persistente

## Objetivo da tela / componente

Refatorar o wizard de nova proposta de 2 para 3 steps bem definidos:

| Step | Label | Conteúdo |
|------|-------|----------|
| 1 | **Dados da Venda** | Empreendimento, Unidade, Cliente |
| 2 | **Comissão** | Corretor, Imobiliária, Taxas de Comissão |
| 3 | **Pagamento** | Índice de Correção, Schedules de Parcelas |

Extrair `<SaleFormSummary>` como componente autônomo e exibi-lo como **sidebar persistente nos 3 steps** — sem duplicação de código.

---

## Fluxo principal do usuário

```
[Nova Proposta]
    │
    ▼
[Step 1 — Dados da Venda]
  - Seleciona Empreendimento → habilita Unidade
  - Seleciona Unidade → Resumo mostra preço da unidade
  - Seleciona Cliente
  - Clica "Avançar" (habilitado somente com Unidade selecionada)
    │
    ▼
[Step 2 — Comissão]
  - Chip read-only: Empreendimento + Unidade
  - Resumo: Preço da Unidade + Valor Financiado R$ 0,00 (schedules ainda não configurados)
  - Seleciona Corretor (opcional) → revela taxa do corretor
  - Seleciona Imobiliária (opcional, depende de Corretor) → revela taxa da imobiliária
  - Clica "Avançar" (sempre habilitado — comissão é opcional)
  - Clica "Voltar" → retorna ao Step 1
    │
    ▼
[Step 3 — Pagamento]
  - Chip read-only: Empreendimento + Unidade
  - Resumo: conteúdo completo (financiado, preço, diferença, término, detalhamento)
  - Seleciona Índice de Correção
  - Configura Schedules de Parcelas (InstallmentScheduleBuilder)
  - Clica "Salvar Proposta" → submit
  - Clica "Voltar" → retorna ao Step 2
    │
    ▼
[Proposta criada — redireciona para lista]
```

## Layout do Grid — Desktop (≥ lg)

Em todos os 3 steps, o layout usa **dois painéis lado a lado**:

```
┌─────────────────────────────────────────┬──────────────────────┐
│  Conteúdo principal (flex-1)            │  <SaleFormSummary>   │
│                                         │  340px               │
│  [Step 1] Empreendimento + Unidade +    │  lg:sticky           │
│           Cliente                       │  lg:top-24           │
│                                         │                      │
│  [Step 2] Chip (read-only) +            │  Resumo Financeiro   │
│           Comissão                      │  (conteúdo varia     │
│                                         │   por step — ver     │
│  [Step 3] Chip (read-only) +            │   seção abaixo)      │
│           Pagamento                     │                      │
└─────────────────────────────────────────┴──────────────────────┘
```

**Classes Tailwind:**
```
grid grid-cols-1 items-start gap-6 lg:grid-cols-[1fr_340px]
```

O `<SaleFormSummary>` recebe o wrapper:
```
<div className="lg:sticky lg:top-24">
  <SaleFormSummary ... />
</div>
```

> **IMPORTANTE:** O grid com 2 colunas se aplica a TODOS os 3 steps. Atualmente o Step 1 não tem esse layout — isso muda nesta story.

## Layout Mobile (< lg)

No breakpoint mobile, o grid colapsa para `grid-cols-1`. O `<SaleFormSummary>` aparece **abaixo do conteúdo principal** em todos os steps.

**Comportamento mobile:**
- Sidebar renderiza abaixo do card de conteúdo
- Não há sticky no mobile (scroll normal)
- Toda a área de Summary é visível rolando a página
- Card de Summary usa largura completa (`w-full`)

**Não há collapse/accordion no mobile** — o Summary é sempre visível (expanded).

## Stepper Visual — 3 Items

O stepper exibe 3 items conectados por linhas separadoras.

```
[✓] Dados da Venda  ─────  [2] Comissão  ─────  [3] Pagamento
 ↑ concluído                ↑ ativo               ↑ futuro
```

### Estados dos step indicators:

| Estado | Visual | Classes |
|--------|--------|---------|
| **Concluído** (step < atual) | Círculo primário + ícone `<Check />` | `bg-primary text-primary-foreground` |
| **Ativo** (step === atual) | Círculo primário + número | `bg-primary text-primary-foreground` |
| **Futuro** (step > atual) | Círculo outline + número | `border-2 border-muted-foreground text-muted-foreground` |

### Label de step:

| Estado | Classe |
|--------|--------|
| Ativo ou concluído | texto padrão (cor normal) |
| Futuro | `text-muted-foreground` |

### Estrutura HTML:

```tsx
<ol className="flex items-center">
  {/* Step 1 */}
  <li className="flex items-center" aria-current={step === 1 ? 'step' : undefined}>
    <div className={cn('flex h-8 w-8 items-center justify-center rounded-full text-sm font-medium',
      step > 1 ? 'bg-primary text-primary-foreground' : 'bg-primary text-primary-foreground'
    )}>
      {step > 1 ? <Check className="size-4" /> : '1'}
    </div>
    <span className="ml-2 text-sm">Dados da Venda</span>
  </li>

  <li className="mx-4 h-px flex-1 bg-border" aria-hidden="true" />

  {/* Step 2 */}
  <li className="flex items-center" aria-current={step === 2 ? 'step' : undefined}>
    <div className={cn('flex h-8 w-8 items-center justify-center rounded-full text-sm font-medium',
      step > 2
        ? 'bg-primary text-primary-foreground'
        : step === 2
          ? 'bg-primary text-primary-foreground'
          : 'border-2 border-muted-foreground text-muted-foreground'
    )}>
      {step > 2 ? <Check className="size-4" /> : '2'}
    </div>
    <span className={cn('ml-2 text-sm', step < 2 && 'text-muted-foreground')}>Comissão</span>
  </li>

  <li className="mx-4 h-px flex-1 bg-border" aria-hidden="true" />

  {/* Step 3 */}
  <li className="flex items-center" aria-current={step === 3 ? 'step' : undefined}>
    <div className={cn('flex h-8 w-8 items-center justify-center rounded-full text-sm font-medium',
      step === 3
        ? 'bg-primary text-primary-foreground'
        : 'border-2 border-muted-foreground text-muted-foreground'
    )}>
      3
    </div>
    <span className={cn('ml-2 text-sm', step < 3 && 'text-muted-foreground')}>Pagamento</span>
  </li>
</ol>
```

## Decisão de Campos por Step

### Step 1 — "Dados da Venda"

**Campos:**
- Empreendimento (`ProjectAutocomplete`)
- Unidade (`UnitAutocomplete`, desabilitada sem Empreendimento)
- **Cliente** (`CustomerAutocomplete`) ← **movido do atual step 2**

**Motivo:** "Dados da Venda" é a identificação completa da transação. Cliente é dado primário da venda, não financeiro.

**Habilitação do botão "Avançar":**
- `canProceed = selectedUnit !== null`
- Cliente NÃO bloqueia o avanço de step (é validado no submit via Zod)
- Label do botão: **"Avançar"** com ícone `ArrowRight`

---

### Step 2 — "Comissão"

**Campos:**
- Chip read-only (Empreendimento + Unidade, com botão "Alterar" → volta ao step 1)
- `broker_id` (Corretor)
- `commission_broker_rate` (aparece somente se corretor selecionado)
- `agency_id` (Imobiliária, desabilitada sem corretor)
- `commission_agency_rate` (aparece somente se imobiliária selecionada)

**Habilitação do botão "Avançar":**
- **Sempre habilitado** — comissão é opcional
- Label: **"Avançar"** com ícone `ArrowRight`

**Botão "Voltar":**
- Label: **"Voltar"** com ícone `ArrowLeft`
- Retorna ao Step 1

---

### Step 3 — "Pagamento"

**Campos:**
- Chip read-only (Empreendimento + Unidade, com botão "Alterar" → volta ao step 1)
- `index_type_code` (Índice de Correção) ← **movido do atual step 2**
- `InstallmentScheduleBuilder` (schedules de parcelas intactos)

**Motivo para `index_type_code` no Step 3:** É um parâmetro diretamente relacionado à correção monetária das parcelas — pertence semanticamente ao contexto de Pagamento.

**Botões:**
- **"Voltar"** (`ArrowLeft`) → retorna ao Step 2
- **"Salvar Proposta"** (`Save`) → submit do formulário (com estado loading)

## Comportamentos Esperados

### Habilitação do botão "Avançar"

| Step | Condição para habilitar |
|------|------------------------|
| Step 1 | `selectedUnit !== null` (projeto também necessário para selecionar unidade) |
| Step 2 | Sempre habilitado (`disabled={false}`) |
| Step 3 | N/A — botão de submit (habilitado conforme Zod validation) |

### Preservação de dados ao navegar

- Navegar Voltar/Avançar entre steps **não perde nenhum dado de formulário**
- `useForm` e `useFieldArray` persistem no `<SaleForm>` pai
- Steps são puros componentes de apresentação — não fazem reset

### Chip read-only (Steps 2 e 3)

Presente no topo dos steps 2 e 3 com:
- Check icon verde + nome do empreendimento + nome + preço da unidade
- Botão "Alterar" (ghost, size="sm") → `setStep(1)` para voltar ao step 1

### Botão "Avançar" Step 1

- Label: **"Avançar"** (não "Próximo" — padronizar com os demais steps)
- Ícone: `ArrowRight` à direita
