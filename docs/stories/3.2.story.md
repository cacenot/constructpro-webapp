## História

**Como** usuário autenticado (vendedor ou gerente),
**Quero** editar os campos de corretor e comissão em propostas existentes e visualizar um resumo de comissão na página de detalhe,
**Para que** eu possa corrigir ou complementar a intenção de comissão em propostas no status `proposal` e consultar os dados de comissão prevista em qualquer proposta que tenha corretor vinculado.

## Critérios de Aceitação

1. **Seção "Comissão" no formulário de edição:** exibida em `sale-edit-form.tsx` com os mesmos quatro campos de Story 3.1 (`broker_id`, `commission_broker_rate`, `agency_id`, `commission_agency_rate`), todos opcionais.
2. **Pré-população:** os campos são pré-populados com os valores atuais da proposta quando a página de edição carrega (`sale.broker?.id`, `sale.agency?.id`, valores de taxa convertidos de PPM → % para exibição).
3. **Read-only quando não editável:** quando `sale.status !== 'proposal'`, os quatro campos de comissão são exibidos como texto (read-only), não como inputs/selects interativos.
4. **Edição — Lógica de seleção:** mesmas regras de habilitação da Story 3.1: Select Imobiliária habilitado somente quando corretor selecionado; taxas aparecem condicionalmente (renderização condicional, não `disabled`).
5. **Edição — Limpeza em cascata:** ao desmarcar corretor, `agency_id`, `commission_broker_rate` e `commission_agency_rate` são resetados para `null`; ao desmarcar imobiliária, `commission_agency_rate` é resetado para `null`.
6. **Edição — Validação Zod:** `saleEditFormSchema` recebe `.superRefine()` com as mesmas regras C1 da Story 3.1.
7. **Submit PATCH:** os quatro campos incluídos no body do `PATCH /api/v1/sales/{sale_id}` usando o helper `buildCommissionRate()` (mesmo padrão do POST em Story 3.1); quando `null`, enviar `null` (permite limpar comissão existente).
8. **Card "Comissão Prevista" no detalhe:** exibido em `pages/vendas/@id/+Page.tsx` somente quando `sale.broker_id != null`; contém:
   - Nome do corretor (`sale.broker?.full_name`)
   - Nome da imobiliária (`sale.agency?.trade_name ?? sale.agency?.legal_name`) — linha visível apenas quando `sale.agency_id != null`
   - Taxa do corretor em % (converter de PPM: `ppm / 10000`)
   - Taxa da imobiliária em % — linha visível apenas quando `sale.agency_id != null`
   - Comissão total estimada em R$ (soma das taxas × valor da venda `sale.amount.cents / 100`)
9. **Propostas sem comissão:** quando `sale.broker_id == null`, o card "Comissão Prevista" não é renderizado (sem estado vazio, sem placeholder).
10. **Lint e build:** `npm run lint` e `npm run build` sem erros ao término.

### `buildCommissionRate()` — helper criado em Story 3.1

Localizado em `pages/vendas/novo/+Page.tsx`. Verificar se vale mover para `src/lib/commission-utils.ts` para reutilização. Se não houver outras chamadas, duplicar inline no `editar/+Page.tsx` é aceitável.

```ts
function buildCommissionRate(percentual: number) {
  const ppm = Math.round(percentual * 10000)
  return {
    ppm,
    percentage: percentual,
    decimal: percentual / 100,
    basis_points: ppm / 100,
    formatted: `${percentual}%`,
  }
}
```

### Enviar `null` para limpar comissão no PATCH

Diferença crítica em relação ao POST (criação): no PATCH, se o usuário limpar o corretor, os campos **devem ser enviados como `null`** (não omitidos). Omitir é no-op na API — a comissão antiga permaneceria.

```ts
// CORRETO — envia null para limpar
broker_id: data.broker_id ?? null,
commission_broker_rate: data.broker_id && data.commission_broker_rate
  ? buildCommissionRate(data.commission_broker_rate)
  : null,
```
