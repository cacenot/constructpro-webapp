## História

**Como** usuário autenticado (vendedor ou gerente),
**Quero** cadastrar, listar, visualizar, editar e remover corretores,
**Para que** possa manter o cadastro de mediadores e vinculá-los a propostas de venda futuramente.

## Critérios de Aceitação

1. **Lista paginada:** página `/corretores` exibe tabela com colunas: nome completo, CPF (mascarado), CRECI, e-mail, telefone e coluna de ações.
2. **Busca server-side:** campo de busca filtra por `full_name`, `cpf` ou `creci` via query param; debounce de 300ms; reset para página 1 ao alterar busca.
3. **Paginação server-side:** mínimo 10 itens por página; exibe info "X–Y de Z corretores"; botões de página funcionais.
4. **Skeleton rows** exibidos durante carregamento inicial e ao mudar filtros/página.
5. **Empty state** com mensagem em português exibido quando a lista está vazia (com e sem filtro ativo).
6. **Criação:** formulário em `/corretores/novo` cria corretor via `POST /api/v1/brokers`; toast de sucesso e redirect para `/corretores` ao concluir.
7. **Detalhe:** página `/corretores/:id` exibe todos os campos do corretor (CPF mascarado, telefone formatado).
8. **Edição:** formulário em `/corretores/:id/editar` atualiza via `PATCH /api/v1/brokers/{id}`; toast de sucesso e redirect para `/corretores/:id` ao concluir.
9. **Soft-delete com confirmação:** botão de excluir abre `AlertDialog` em português antes de chamar `DELETE /api/v1/brokers/{id}`; toast de sucesso e redirect para `/corretores` ao concluir.
10. **Validação Zod (broker.schema.ts):**
    - `cpf`: CPF válido (usar `validateCPF` do api-client), campo obrigatório.
    - `full_name`: 3–120 caracteres, obrigatório.
    - `creci`: 5–20 caracteres, obrigatório.
    - `email`: e-mail válido, opcional.
    - `phone`: formato E.164 (stripped de máscara), opcional.
11. **Máscara CPF:** reutilizar input/máscara já existente no módulo `customers` (sem duplicar código).
12. **Navegação:** entrada "Corretores" adicionada ao grupo "Comercial" em `src/components/app-sidebar.tsx`; link ativo destacado ao acessar `/corretores`.
13. **Responsividade:** colunas secundárias (e-mail, telefone) ocultas em mobile (`< md`).
14. **Tooltips:** todos os botões de ícone possuem `<Tooltip>` com label em português (ex.: "Editar", "Excluir", "Ver detalhes").
15. **Dropdown de ações:** `DropdownMenuLabel` com "Ações" no topo; `DropdownMenuSeparator` entre grupos; itens sem ícone.
16. **Error handling:** mutations usam `throwApiError` no `mutationFn` e `handleApiError` no `onError` / `catch`.
17. **Testes unitários:** `npx vitest run src/schemas/broker.schema.test.ts src/hooks/use-brokers-table.test.ts` passa 100% — BLOQUEANTE.
18. **Testes E2E:** `npx playwright test e2e/corretores/` passa 100% (cenários: lista, criação, edição, soft-delete com confirmação, busca) — BLOQUEANTE.
19. **Build e lint:** `npm run build` e `npm run lint` sem erros/warnings ao término.

### API — Broker

[Source: DIRECIONAMENTO-FRONTEND.md §4, docs/stories/epic-2-corretores-imobiliarias.md]

**Endpoints:**

| Método | Endpoint | Uso |
|--------|----------|-----|
| `POST` | `/api/v1/brokers` | Criar corretor |
| `GET` | `/api/v1/brokers` | Lista paginada + busca |
| `GET` | `/api/v1/brokers/{id}` | Detalhe |
| `PATCH` | `/api/v1/brokers/{id}` | Editar |
| `DELETE` | `/api/v1/brokers/{id}` | Soft-delete |

**Campos do broker:**

| Campo | Tipo | Validação | Obrigatório |
|-------|------|-----------|-------------|
| `cpf` | string | único, `validateCPF` do api-client | sim |
| `full_name` | string | 3–120 chars | sim |
| `creci` | string | 5–20 chars, único | sim |
| `email` | string | e-mail válido | não |
| `phone` | string | E.164 (stripped de máscara) | não |

**Padrão de chamada API** (source: `pages/clientes/novo/+Page.tsx`):

```ts
const { client } = useApiClient()
const createMutation = useMutation({
  mutationFn: async (data) => {
    const { data: response, error } = await client.POST('/api/v1/brokers', {
      body: { ...data, cpf: data.cpf.replace(/\D/g, '') },
    })
    if (error) throwApiError(error, 'Falha ao cadastrar corretor')
    return response
  },
  onSuccess: () => { toast.success('Corretor cadastrado com sucesso!'); navigate('/corretores') },
  onError: (error) => handleApiError(error, 'Falha ao cadastrar corretor'),
})
```
