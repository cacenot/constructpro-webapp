## História

**Como** usuário autenticado (vendedor ou gerente),
**Quero** cadastrar, listar, visualizar, editar e remover imobiliárias,
**Para que** possa manter o cadastro de agências parceiras e vinculá-las a propostas de venda futuramente.

## Critérios de Aceitação

1. **Lista paginada:** página `/imobiliarias` exibe tabela com colunas: razão social, nome fantasia, CNPJ (mascarado), CRECI-J, e-mail e coluna de ações.
2. **Busca server-side:** campo de busca filtra por `legal_name`, `trade_name`, `cnpj` ou `creci_j` via query param; debounce de 300ms; reset para página 1 ao alterar busca.
3. **Paginação server-side:** mínimo 10 itens por página; exibe info "X–Y de Z imobiliárias"; botões de página funcionais.
4. **Skeleton rows** exibidos durante carregamento inicial e ao mudar filtros/página.
5. **Empty state** com mensagem em português exibido quando a lista está vazia (com e sem filtro ativo).
6. **Criação:** formulário em `/imobiliarias/novo` cria imobiliária via `POST /api/v1/agencies`; toast de sucesso e redirect para `/imobiliarias` ao concluir.
7. **Detalhe:** página `/imobiliarias/:id` exibe todos os campos da imobiliária (CNPJ mascarado, telefone formatado, nome fantasia se presente).
8. **Edição:** formulário em `/imobiliarias/:id/editar` atualiza via `PATCH /api/v1/agencies/{id}`; toast de sucesso e redirect para `/imobiliarias/:id` ao concluir.
9. **Soft-delete com confirmação:** botão de excluir abre `AlertDialog` em português antes de chamar `DELETE /api/v1/agencies/{id}`; toast de sucesso e redirect para `/imobiliarias` ao concluir.
10. **Validação Zod (agency.schema.ts):**
    - `cnpj`: CNPJ válido (usar `validateCNPJ` do api-client), campo obrigatório.
    - `legal_name`: 3–160 caracteres, obrigatório.
    - `trade_name`: 0–120 caracteres, opcional.
    - `creci_j`: 5–20 caracteres, obrigatório.
    - `email`: e-mail válido, opcional.
    - `phone`: formato E.164 (stripped de máscara), opcional.
11. **Máscara CNPJ:** reutilizar input/máscara já existente no módulo `customers` (sem duplicar código).
12. **Navegação:** entrada "Imobiliárias" adicionada ao grupo "Comercial" em `src/components/app-sidebar.tsx`; link ativo destacado ao acessar `/imobiliarias`.
13. **Responsividade:** colunas secundárias (e-mail, nome fantasia) ocultas em mobile (`< md`).
14. **Tooltips:** todos os botões de ícone possuem `<Tooltip>` com label em português (ex.: "Editar", "Excluir", "Ver detalhes").
15. **Dropdown de ações:** `DropdownMenuLabel` com "Ações" no topo; `DropdownMenuSeparator` entre grupos; itens sem ícone.
16. **Error handling:** mutations usam `throwApiError` no `mutationFn` e `handleApiError` no `onError` / `catch`.
17. **Testes unitários:** `npx vitest run src/schemas/agency.schema.test.ts src/hooks/use-agencies-table.test.ts` passa 100% — BLOQUEANTE.
18. **Testes E2E:** ~~`npx playwright test e2e/imobiliarias/`~~ — **DEFERIDO** como débito técnico `[EPIC2-T1]`. Spec criada em `tests/e2e/imobiliarias/agency-crud.spec.ts`, execução pendente. Ver `docs/STORY-BACKLOG.md`.

### API — Agency

[Source: docs/stories/epic-2-corretores-imobiliarias.md — Campos da Imobiliária e Endpoints Agency]

**Endpoints:**

| Método | Endpoint | Uso |
|--------|----------|-----|
| `POST` | `/api/v1/agencies` | Criar imobiliária |
| `GET` | `/api/v1/agencies` | Lista paginada + busca |
| `GET` | `/api/v1/agencies/{id}` | Detalhe |
| `PATCH` | `/api/v1/agencies/{id}` | Editar |
| `DELETE` | `/api/v1/agencies/{id}` | Soft-delete |

**Campos da agency:**

| Campo | Tipo | Validação | Obrigatório |
|-------|------|-----------|-------------|
| `cnpj` | string | único, `validateCNPJ` do api-client | sim |
| `legal_name` | string | 3–160 chars | sim |
| `trade_name` | string | 0–120 chars | não |
| `creci_j` | string | 5–20 chars, único | sim |
| `email` | string | e-mail válido | não |
| `phone` | string | E.164 (stripped de máscara) | não |

**Parâmetros de busca (GET /api/v1/agencies):**
```
?search=<term>   # filtra por legal_name, trade_name, cnpj, creci_j
&page=<n>
&page_size=<n>
```

**Padrão de chamada API** (baseado em `pages/corretores/novo/+Page.tsx`):

```ts
const { client } = useApiClient()
const createMutation = useMutation({
  mutationFn: async (data) => {
    const { data: response, error } = await client.POST('/api/v1/agencies', {
      body: { ...data, cnpj: data.cnpj.replace(/\D/g, '') },
    })
    if (error) throwApiError(error, 'Falha ao cadastrar imobiliária')
    return response
  },
  onSuccess: () => { toast.success('Imobiliária cadastrada com sucesso!'); navigate('/imobiliarias') },
  onError: (error) => handleApiError(error, 'Falha ao cadastrar imobiliária'),
})
```

### Schema Zod — agency.schema.ts

[Source: docs/stories/epic-2-corretores-imobiliarias.md, broker.schema.ts (Story 2.2)]

```ts
export const agencyCreateSchema = z.object({
  cnpj: z.string().refine((val) => validateCNPJ(val.replace(/\D/g, '')), { message: 'CNPJ inválido' }),
  legal_name: z.string().min(3, 'Mínimo 3 caracteres').max(160, 'Máximo 160 caracteres'),
  trade_name: z.string().max(120, 'Máximo 120 caracteres').optional().or(z.literal('')),
  creci_j: z.string().min(5, 'Mínimo 5 caracteres').max(20, 'Máximo 20 caracteres'),
  email: z.string().email('E-mail inválido').optional().or(z.literal('')),
  phone: z.string().refine(
    (val) => !val || isValidPhoneNumber(val),
    { message: 'Telefone inválido' }
  ).optional().or(z.literal('')),
})

export const agencyUpdateSchema = agencyCreateSchema.omit({ cnpj: true }).partial()

export type AgencyCreateFormData = z.infer<typeof agencyCreateSchema>
export type AgencyUpdateFormData = z.infer<typeof agencyUpdateSchema>
```
