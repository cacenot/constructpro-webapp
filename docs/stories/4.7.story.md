## Story

**As a** corretor preenchendo o formulário de nova proposta,
**I want** atribuir índices de correção diferentes para cada grupo de parcelas — ou usar um único índice para toda a proposta com um toggle —,
**so that** propostas com grupos de parcelas em índices distintos (ex: parcelas regulares em IGPM, entrega das chaves em INCC) sejam registradas corretamente sem workarounds manuais.

## Acceptance Criteria

1. No Step 3 ("Pagamento"), acima do `<InstallmentScheduleBuilder>`, existe um toggle **"Usar o mesmo índice para toda a proposta"**. Por padrão está **ativado**.
2. Com toggle **ativado**: o seletor global `index_type_code` permanece no topo do Step 3 (comportamento idêntico ao atual). Campo é obrigatório. Cada schedule no payload recebe o valor global.
3. Com toggle **desativado**: o seletor global desaparece. Cada grupo de parcelas no `<InstallmentScheduleBuilder>` exibe seu próprio campo **"Índice de Correção"** (obrigatório), imediatamente acima dos campos do grupo.
4. Quando toggle é alternado de ativado → desativado, o valor do seletor global é pré-preenchido em todos os grupos já adicionados. Quando alternado de desativado → ativado, o seletor global recebe o índice do primeiro grupo (ou fica vazio se não houver grupos).
5. Schema Zod: `installmentScheduleSchema` recebe campo `index_type_code?: string | null`. `saleFormSchema.index_type_code` torna-se obrigatório somente quando toggle está ativado. `superRefine` no schema do schedule valida que `index_type_code` é preenchido quando toggle está desativado.
6. Payload enviado à API:
   - Toggle **on** → `{ index_type_code: "GLOBAL", installment_schedules: [...schedules sem index_type_code] }` (comportamento atual)
   - Toggle **off** → `{ installment_schedules: [...schedules com index_type_code por item] }` — sem `index_type_code` global. ⚠️ Contrato exato aguarda spec da API #93.
7. `sale-form-step3.tsx`: toggle como `useState<boolean>(true)` local no Step 3; valor do toggle e do índice global passados via props para o `<InstallmentScheduleBuilder>` quando toggle off.
8. `sale-edit-form.tsx` recebe a mesma lógica de toggle + seletor por grupo.
9. Sem regressão: propostas com toggle on e índice global têm payload idêntico ao atual — nenhum campo existente alterado.
10. Lint (`npm run lint`) e build (`npm run build`) limpos.
