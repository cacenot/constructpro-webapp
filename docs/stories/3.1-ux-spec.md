# UX Spec — Story 3.1: Comissão na Criação de Proposta

## Objetivo da tela / componente

Adicionar um **Card "Comissão"** ao formulário de Nova Proposta (`sale-form.tsx`) que permite ao usuário selecionar corretor e/ou imobiliária, vinculando taxas de comissão em percentual.

O card é **brownfield enhancement** — inserido na left column entre o Card "Dados da Venda" (existente) e o Card "Pagamento" (existente). Todos os campos são opcionais; a ausência de comissão não afeta o fluxo normal de criação de proposta.

O objetivo da experiência é tornar a intenção de comissão **explícita desde a criação**, com regras visuais claras que espelham as regras de negócio da API (C1: broker e taxa sempre juntos; agency requer broker).

---

## Fluxo principal do usuário

```
Usuário acessa Nova Proposta
│
├─ Preenche Dados da Venda (existente)
│
├─ [Card Comissão — novo]
│   ├─ (A) Sem comissão → ignora o card e segue para Pagamento
│   │
│   ├─ (B) Apenas corretor
│   │   ├─ Seleciona corretor no Select Corretor
│   │   ├─ Campo "Taxa Corretor (%)" aparece
│   │   ├─ Informa a taxa (ex.: 1.5)
│   │   └─ Segue para Pagamento
│   │
│   └─ (C) Corretor + imobiliária
│       ├─ Seleciona corretor → Select Imobiliária habilita
│       ├─ Campo "Taxa Corretor (%)" aparece → informa taxa
│       ├─ Seleciona imobiliária
│       ├─ Campo "Taxa Imobiliária (%)" aparece → informa taxa
│       └─ Segue para Pagamento
│
└─ Submete proposta → API recebe campos opcionais (ou omite se vazios)
```

**Regra de cascata (desfazer):**
- Ao limpar o Select Corretor → Select Imobiliária volta ao estado desabilitado, ambas as taxas somem e seus valores são resetados para `null`.
- Ao limpar o Select Imobiliária → somente Taxa Imobiliária some e é resetada.

## Comportamentos esperados

### B1 — Aparecimento condicional das taxas

- **Não usar `disabled` para ocultar taxas** — usar renderização condicional (`{watchedBrokerId && <FormField ... />}`)
- Ao aparecer, o campo de taxa recebe foco automaticamente somente se o corretor foi selecionado via interação manual (não em re-renders de formulário)
- Transição visual: sem animação — aparecimento imediato (sem uso de framer-motion)

### B2 — Habilitação do Select Imobiliária

- Prop `disabled={!watchedBrokerId}` no `<SelectTrigger>`
- `aria-disabled="true"` quando desabilitado (acessibilidade)
- Placeholder muda conforme estado:
  - Sem broker: "Selecione um corretor primeiro"
  - Com broker: "Selecione uma imobiliária"

### B3 — Limpeza em cascata ao limpar Corretor

Ao detectar `broker_id` retornando a `null`/`undefined` (via `onValueChange` do Select Corretor):

```ts
form.setValue('agency_id', null)
form.setValue('commission_broker_rate', null)
form.setValue('commission_agency_rate', null)
```

Usar `useWatch` para detectar a mudança e acionar o reset.

### B4 — Limpeza em cascata ao limpar Imobiliária

Ao limpar `agency_id`:

```ts
form.setValue('commission_agency_rate', null)
```

### B5 — Input de taxa: formato e experiência

- `type="number"`, `step="0.01"`, `min="0"`, sem `max` (API valida cap via 422)
- Placeholder: `"Ex: 1.5"` para taxa corretor; `"Ex: 3.0"` para taxa imobiliária
- Exibe percentual (%) — conversão para PPM transparente, ocorre apenas no `mutationFn`
- Não formatar automaticamente — usuário digita `1.5`, não `1,5` (campo número nativo)

### B6 — Labels e textos em português

| Campo | Label | Placeholder habilitado | Placeholder desabilitado |
|-------|-------|----------------------|------------------------|
| broker_id | Corretor | Selecione um corretor | — |
| commission_broker_rate | Taxa de Comissão do Corretor (%) | Ex: 1.5 | — |
| agency_id | Imobiliária | Selecione uma imobiliária | Selecione um corretor primeiro |
| commission_agency_rate | Taxa de Comissão da Imobiliária (%) | Ex: 3.0 | — |

### B7 — Mensagens de erro Zod (português)

| Regra | Mensagem |
|-------|---------|
| broker sem taxa | "Taxa de comissão do corretor é obrigatória quando corretor selecionado." |
| taxa ≤ 0 | "Taxa deve ser maior que zero." |
| agency sem broker | "Selecione um corretor antes de escolher imobiliária." |
| agency sem taxa | "Taxa de comissão da imobiliária é obrigatória quando imobiliária selecionada." |

### B8 — Posicionamento do Card

**Left column** do grid `[1fr_340px]`, **entre** Card "Dados da Venda" e Card "Pagamento":

```
Left column
├── Card "Dados da Venda"     ← existente (não alterar)
├── Card "Comissão"           ← NOVO (story 3.1)
└── Card "Pagamento"          ← existente (não alterar)
```

## Referências

### Design system / tokens

- **Espaçamento:** `space-y-4` dentro do `CardContent`, `gap-4` entre colunas — idêntico ao Card "Dados da Venda"
- **Grid interno:** `sm:grid-cols-12` com `grid gap-4` — mesmo padrão do restante do form
- **Tipografia:** `FormLabel` (shadcn) para labels; `text-sm text-muted-foreground` para hints — sem novos estilos
- **Cores:** estado de erro via `FormMessage` (herda `text-destructive` do design system)

### Componentes shadcn a usar

| Componente | Uso |
|-----------|-----|
| `Card`, `CardHeader`, `CardTitle`, `CardContent` | Container da seção |
| `FormField`, `FormItem`, `FormLabel`, `FormControl`, `FormMessage` | Todos os campos |
| `Select`, `SelectTrigger`, `SelectValue`, `SelectContent`, `SelectItem` | Selects de broker e agency |
| `Input` | Inputs de taxa (type="number") |

Todos já importados no projeto. Não adicionar novos pacotes.

### Padrão canônico de Select com FK (reutilizar)

O campo `index_type_code` em `sale-form.tsx` (linhas 66–78) demonstra o padrão exato:

```ts
const xQuery = useQuery({
  queryKey: ['x-key'],
  queryFn: async () => {
    const { data, error } = await client.GET('/api/v1/endpoint/', {
      params: { query: { page: 1, page_size: 100 } },
    })
    if (error) throw new Error('Falha ao carregar X')
    return data
  },
  staleTime: 5 * 60 * 1000,
})
```

Replicar para `brokersQuery` (endpoint `/api/v1/brokers`) e `agenciesQuery` (endpoint `/api/v1/agencies`).

### Ícones

N/A — nenhum novo ícone. Não adicionar ícone ao `CardTitle` "Comissão" (inconsistente com os demais cards do form que também não usam ícone).

### Acessibilidade

- `aria-disabled="true"` no `SelectTrigger` de Imobiliária quando desabilitado
- `disabled` prop nativa (não apenas CSS) para garantir que o campo não receba foco via teclado
- `FormLabel` associado corretamente ao controle via `FormField` do shadcn (htmlFor automático)
- Mensagens de erro (`FormMessage`) são lidas por screen readers via `aria-live` (comportamento padrão do shadcn Form)
