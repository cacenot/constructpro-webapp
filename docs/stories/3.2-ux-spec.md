# UX Spec — Story 3.2: Comissão na Edição e Exibição de Proposta

## Objetivo da tela / componente

Esta story cobre **dois artefatos distintos**:

1. **Card "Comissão" no formulário de edição** (`sale-edit-form.tsx`) — brownfield enhancement que replica a seção da Story 3.1 com duas diferenças críticas: os campos são **pré-populados** com os valores atuais da proposta e exibidos como **texto read-only** quando `sale.status !== 'proposal'`.

2. **Card "Comissão Prevista" na página de detalhe** (`pages/vendas/@id/+Page.tsx`) — card informativo novo, condicional à existência de `sale.broker_id`, que resume corretor, imobiliária e comissão total estimada em reais.

O objetivo da experiência é:
- No formulário de edição: permitir corrigir a intenção de comissão enquanto a proposta ainda é editável, sem surpresas de estado.
- Na página de detalhe: oferecer visibilidade imediata da comissão prevista sem poluir a UI quando não há corretor.

---

## Fluxo principal do usuário

### Fluxo A — Edição de proposta com comissão

```
Usuário acessa /vendas/{id}/editar
│
├─ [status = proposal] → forma editável
│   ├─ Card "Comissão" exibe selects + inputs pré-populados com valores da API
│   │
│   ├─ (A1) Proposta sem comissão existente → campos em branco, mesma UX de 3.1
│   │
│   ├─ (A2) Proposta com corretor → broker select pré-selecionado, taxa pré-preenchida
│   │   ├─ Agency select habilitado se agência existir
│   │   └─ Taxa agência pré-preenchida se agência existir
│   │
│   ├─ (A3) Usuário limpa corretor → cascata: agency + taxas → null
│   │
│   └─ Submit → PATCH com campos (null = limpar intenção existente)
│
└─ [status ≠ proposal] → forma read-only
    ├─ Alert "Edição não disponível" já existente no topo (inalterado)
    └─ Card "Comissão" exibe valores como texto (padrão do Card "Dados da Venda")
        ├─ Com comissão: exibe nome do corretor, taxa, imobiliária (se houver), taxa
        └─ Sem comissão: exibe "—" em cada campo
```

### Fluxo B — Visualização de detalhe de proposta

```
Usuário acessa /vendas/{id}
│
├─ [sale.broker_id == null] → card "Comissão Prevista" NÃO renderizado
│
└─ [sale.broker_id != null] → card "Comissão Prevista" renderizado
    ├─ Sempre: corretor + taxa corretor + total estimado
    └─ Condicional [sale.agency_id != null]: imobiliária + taxa imobiliária
```

## Comportamentos esperados

### B1 — Posicionamento do Card "Comissão" no formulário de edição

**Entre** Card "Configuração" e Card "Pagamento":

```
Form
├── Card "Dados da Venda"    ← existente, read-only (inalterado)
├── Card "Configuração"      ← existente (Select índice de correção)
├── Card "Comissão"          ← NOVO (story 3.2)
└── Card "Pagamento"         ← existente (entrada + parcelas)
```

### B2 — Posicionamento do Card "Comissão Prevista" na página de detalhe

**Nova row abaixo** da section "Values + Contract", **antes** do cronograma de parcelas:

```
{/* Info cards — 3-col */}   (inalterado)
{/* Values + Contract — 2-col */}  (inalterado)
{sale.broker_id && (
  <Card> ... Comissão Prevista ... </Card>   ← NOVA ROW (full width)
)}
{/* Cronograma de Parcelas */}   (inalterado)
```

Razão: expandir o grid existente para 3 colunas quebraria o equilíbrio visual dos cards densos "Valores da Proposta" e "Contrato". Card em row separada é mais limpa e não requer lógica de grid dinâmico.

### B3 — Read-only no formulário: padrão de texto

Reutilizar o padrão já existente no Card "Dados da Venda" (`sale-edit-form.tsx`, linhas 224–239):

```tsx
// padrão canônico de read-only no sale-edit-form
<p className="text-sm text-muted-foreground">Label</p>
<p className="text-sm font-medium">{valor ?? '—'}</p>
```

- Taxas read-only: exibir como `"1,50%"` — converter PPM → % e formatar com `toFixed(2)` + `'%'`
- Sem inputs, sem selects quando `!isEditable`

### B4 — Clearable Select: "Nenhum corretor"

Resolução da observação QA Story 3.1 (LOW UX-001):

- Adicionar `<SelectItem value="">Nenhum corretor</SelectItem>` como **primeiro item** no Select Corretor
- Quando `onValueChange` receber `""`, executar:
  ```ts
  form.setValue('broker_id', null)
  form.setValue('agency_id', null)
  form.setValue('commission_broker_rate', null)
  form.setValue('commission_agency_rate', null)
  ```
- O Select Imobiliária **não** precisa de item "Nenhum" separado — ao limpar o broker, ela já é resetada pela cascata
- Adicionar `<SelectItem value="">Nenhuma imobiliária</SelectItem>` como primeiro item no Select Imobiliária (para o caso de agency ser pré-selecionada e o usuário querer limpar apenas a agency sem limpar o broker)

### B5 — Pré-população via defaultValues

Dentro do `useForm`, expandir `defaultValues` com:

```ts
broker_id: sale.broker?.id ?? null,
commission_broker_rate: sale.commission_broker_rate?.ppm
  ? sale.commission_broker_rate.ppm / 10000
  : null,
agency_id: sale.agency?.id ?? null,
commission_agency_rate: sale.commission_agency_rate?.ppm
  ? sale.commission_agency_rate.ppm / 10000
  : null,
```

- Conversão PPM → %: `ppm / 10000` (ex: `15000 ppm → 1.5%`)
- `null` quando campo não existe na resposta da API

### B6 — Aparecimento condicional das taxas (idêntico a 3.1)

- Taxa Corretor: `{watchedBrokerId && <FormField ... />}` — unmounted, não disabled
- Taxa Imobiliária: `{watchedAgencyId && <FormField ... />}` — unmounted, não disabled
- `watchedBrokerId = useWatch({ control: form.control, name: 'broker_id' })`
- `watchedAgencyId = useWatch({ control: form.control, name: 'agency_id' })`

### B7 — Habilitação do Select Imobiliária (idêntico a 3.1)

- `disabled={!isEditable || !watchedBrokerId}` no `<Select>`
- `aria-disabled="true"` no `<SelectTrigger>` quando desabilitado
- Placeholder dinâmico:
  - Sem broker: `"Selecione um corretor primeiro"`
  - Com broker: `"Selecione uma imobiliária"`

### B8 — Cascata ao limpar Corretor

Via `onValueChange` do Select Corretor, ao receber `""`:

```ts
form.setValue('broker_id', null)
form.setValue('agency_id', null)
form.setValue('commission_broker_rate', null)
form.setValue('commission_agency_rate', null)
```

### B9 — Cascata ao limpar Imobiliária

Via `onValueChange` do Select Imobiliária, ao receber `""`:

```ts
form.setValue('agency_id', null)
form.setValue('commission_agency_rate', null)
```

### B10 — Cálculo do total estimado no card de detalhe

```ts
const brokerRatePct = (sale.commission_broker_rate?.ppm ?? 0) / 10000
const agencyRatePct  = (sale.commission_agency_rate?.ppm  ?? 0) / 10000
const totalRatePct   = brokerRatePct + agencyRatePct
const estimatedCommission = (totalRatePct / 100) * (sale.amount.cents / 100)
// Exibir: formatCurrency(estimatedCommission)
```

Exibição das taxas no card de detalhe:

```ts
// Taxa corretor
`${(sale.commission_broker_rate.ppm / 10000).toFixed(2)}%`
// Taxa agência
`${(sale.commission_agency_rate.ppm / 10000).toFixed(2)}%`
```

## Referências

### Design system / tokens

- **Espaçamento:** `space-y-4` dentro do `CardContent`, `gap-4` entre colunas
- **Grid interno do formulário:** `sm:grid-cols-12` com colunas `sm:col-span-6` (selects) e `sm:col-span-4` (inputs de taxa)
- **Tipografia read-only:** `text-sm text-muted-foreground` (label) + `text-sm font-medium` (valor)
- **Card detalhe layout interno:** `space-y-3` + `flex items-center justify-between` (mesmo padrão de "Valores da Proposta")
- **Separador antes do total:** `<Separator />` (mesmo padrão do Card "Valores da Proposta")
