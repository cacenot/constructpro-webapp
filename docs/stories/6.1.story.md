## Story

**As a** administrador ou gestor comercial,
**I want** visualizar uma listagem paginada de comissões filtrada por corretor, imobiliária e período, com cards de totais no topo,
**so that** eu possa acompanhar o volume de comissões pagas a cada corretor e imobiliária em qualquer período, sem precisar abrir cada venda individualmente.

## Acceptance Criteria

1. A página `/comissoes` exibe cards de totais no topo com os valores:
   - **Total Corretor** — soma de `broker_amount` de todos os registros da query atual
   - **Total Imobiliária** — soma de `agency_amount` de todos os registros da query atual
   - **Total Geral** — soma de `total_amount` de todos os registros da query atual

2. Os cards de totais usam `tabular-nums` e `formatCurrency` (padrão `financeiro/installments-summary-cards.tsx`). Em loading, exibem skeleton idêntico ao padrão.

3. A tabela lista comissões com as seguintes colunas:
   - **Data** — `created_at` formatado como `dd/MM/yyyy` (coluna primária, sempre visível)
   - **Venda** — `sale_id` ou link para a venda (`/vendas/{sale_id}`), sempre visível
   - **Corretor** — `broker.full_name` ou `—` quando `broker_id` é nulo, sempre visível
   - **Imobiliária** — `agency.name` ou `—` quando `agency_id` é nulo, oculta em mobile (`hidden md:table-cell`)
   - **Valor Venda** — `sale_amount_at_approval` em BRL com `tabular-nums`, oculta em mobile
   - **Taxa Corretor** — `broker_rate` como porcentagem (`X,XX%`), oculta em mobile (`hidden lg:table-cell`)
   - **Taxa Imob.** — `agency_rate` como porcentagem, oculta em mobile (`hidden lg:table-cell`)
   - **Comissão Total** — `total_amount` em BRL com `tabular-nums`, sempre visível, alinhado à direita

4. Filtros server-side disponíveis:
   - **Corretor** — combobox com busca por nome (popula `broker_id` na query)
   - **Imobiliária** — combobox com busca por nome (popula `agency_id` na query)
   - **Período** — date range picker com presets `thisMonth`, `lastMonth`, `thisYear` (popula `from` / `to` na query)

5. Paginação server-side: 10 itens por página. Componente exibe "X–Y de Z comissões".

6. Estado vazio: quando não há registros para os filtros aplicados, exibir mensagem "Nenhuma comissão encontrada" com botão "Limpar filtros" (se `hasActiveFilters`).

7. O item **"Comissões"** é adicionado ao grupo `navComercial` em `src/components/app-sidebar.tsx`, após "Imobiliárias". Ícone: `ReceiptText` (lucide-react). Rota: `/comissoes`.

8. A página usa o padrão `AppLayout` e segue a mesma estrutura da página `/financeiro` (header h1 + p de descrição → cards de totais → filtros → Card com tabela + paginação).

9. `npm run build` limpo e `npm run lint` sem warnings após implementação.

10. Nenhuma regressão nas páginas existentes (corretores, imobiliárias, financeiro, vendas).

### Endpoint (API #95 ✅ entregue — confirmar contrato exato no client)

```
GET /api/v1/commissions?broker_id=…&agency_id=…&from=…&to=…&page=…&page_size=…
```

**Response esperada (verificar contrato real no client após `npm install`):**
```typescript
{
  items: CommissionResponse[]    // lista paginada
  total: number                  // total de registros para paginação
  summary?: {                    // agregados — estrutura a confirmar com API
    total_broker_amount: MoneyResponse
    total_agency_amount: MoneyResponse
    total_commission_amount: MoneyResponse
  }
}
```

**CommissionResponse disponível no client (Epic 3):**
```typescript
{
  sale_id: string
  broker_id?: string
  agency_id?: string
  broker_rate?: number           // porcentagem (ex: 1.5 = 1,5%)
  agency_rate?: number
  broker_amount?: MoneyResponse  // { cents, decimal, brl }
  agency_amount?: MoneyResponse
  total_amount: MoneyResponse
  sale_amount_at_approval: MoneyResponse
  created_at: string             // ISO 8601
  broker?: { id, full_name, ... }
  agency?: { id, name, ... }
}
```

**Nota:** Verificar com API #95 se a resposta de listagem inclui `broker` e `agency` aninhados ou apenas os IDs. Se apenas IDs, columns precisarão buscar nomes por lookup adicional.

### Hook use-commissions-table.ts

Seguir padrão canônico de `use-brokers-table.ts` + `use-installments-table.ts`:
- `parseAsString` / `parseAsInteger` via `nuqs` para filtros na URL
- `useQueryStates` com `history: 'push'`
- debounce 300ms apenas se houver campo de busca por texto
- `queryKey: ['commissions', queryParams]` — chave estável como array
- `PAGE_SIZE = 10`

### Cards de Totais

Se o endpoint retornar `summary` com os totais: usar diretamente.
Se o endpoint **não** retornar `summary`: somar `broker_amount.cents`, `agency_amount.cents`, `total_amount.cents` dos `items` retornados.

**Preferência:** consumir `summary` do endpoint (mais preciso para totais globais da query, não apenas da página atual).
