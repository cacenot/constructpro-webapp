# UX Spec — Story 6.1: Comissões — Listagem por corretor, imobiliária e período

## Objetivo da tela / componente

Nova página `/comissoes` que exibe um relatório paginado de comissões geradas por vendas aprovadas. A tela permite ao administrador ou gestor comercial consultar comissões filtradas por corretor, imobiliária e período, com visibilidade imediata dos totais consolidados no topo.

A experiência é modeled exatamente na página `/financeiro` — padrão canônico do projeto: header descritivo → cards de totais → filtros → tabela paginada com estado vazio. Nenhum padrão novo é introduzido.

---

## Fluxo principal do usuário

```
Usuário acessa /comissoes via sidebar (Comercial → Comissões)
│
├─ [Carregando] → 3 cards skeleton + tabela skeleton
│
├─ [Sem filtros] → totais globais + lista completa paginada
│   ├─ Usuário aplica filtro Corretor (combobox)
│   │   └─ Query atualiza: broker_id → página 1 → dados filtrados
│   ├─ Usuário aplica filtro Imobiliária (combobox)
│   │   └─ Query atualiza: agency_id → página 1 → dados filtrados
│   ├─ Usuário aplica filtro Período (date range picker)
│   │   └─ Query atualiza: from + to → página 1 → dados filtrados
│   └─ Usuário combina múltiplos filtros → query acumulativa
│
├─ [Com resultados] → cards com totais da query atual + tabela paginada
│   ├─ Usuário navega entre páginas
│   │   └─ Query atualiza: page → lista atualizada
│   └─ Usuário clica no link da venda (coluna "Venda")
│       └─ Navega para /vendas/{sale_id}
│
└─ [Sem resultados para filtros aplicados] → estado vazio
    └─ Botão "Limpar filtros" → remove todos os filtros → reload
```

## Comportamentos esperados

### B1 — Layout da página (estrutura canônica)

Estrutura idêntica à `/financeiro`:

```tsx
<AppLayout>
  <div className="space-y-6">
    <div>
      <h1 className="text-3xl font-bold tracking-tight">Comissões</h1>
      <p className="mt-2 text-muted-foreground">
        Acompanhe comissões por corretor, imobiliária e período.
      </p>
    </div>

    <CommissionSummaryCards summary={summary} isLoading={isLoading} />

    <CommissionFilters {...filters} />

    <Card className="rounded-xl shadow-sm">
      <CardContent className="p-0">
        <CommissionTable ... />
      </CardContent>
      <CommissionPagination {...pagination} />
    </Card>
  </div>
</AppLayout>
```

### B2 — Cards de totais (3 cards, grid-cols-3)

Grid com 3 colunas (diferente do financeiro que usa 4). Sem `lg:grid-cols-4`:

```
grid-cols-1 sm:grid-cols-3
```

| Card | Label | Dado | Ícone |
|------|-------|------|-------|
| 1 | Total Corretor | `summary.total_broker_amount` | `Handshake` |
| 2 | Total Imobiliária | `summary.total_agency_amount` | `Building2` |
| 3 | Total Geral | `summary.total_commission_amount` | `ReceiptText` |

Se `summary` for `null` (endpoint não retornou): somar `broker_amount.cents` / `agency_amount.cents` / `total_amount.cents` dos `items` (fallback apenas para a página atual — menos preciso).

Valores exibidos com `formatCurrency(cents / 100)` e `tabular-nums`. Skeleton em loading: `Skeleton h-4 w-24` (label) + `Skeleton h-7 w-32` (valor) + `Skeleton h-4 w-4 rounded` (ícone).

### B3 — Colunas da tabela

| # | Coluna | Campo | Visibilidade | Alinhamento |
|---|--------|-------|-------------|-------------|
| 1 | Data | `created_at` (dd/MM/yyyy) | sempre | esquerda |
| 2 | Venda | link `/vendas/{sale_id}` | sempre | esquerda |
| 3 | Corretor | `broker.full_name` ou `—` | sempre | esquerda |
| 4 | Imobiliária | `agency.name` ou `—` | `hidden md:table-cell` | esquerda |
| 5 | Valor Venda | `sale_amount_at_approval` BRL | `hidden md:table-cell` | direita |
| 6 | Taxa Corretor | `broker_rate` % | `hidden lg:table-cell` | direita |
| 7 | Taxa Imob. | `agency_rate` % | `hidden lg:table-cell` | direita |
| 8 | Comissão Total | `total_amount` BRL | sempre | direita |

**Formatação das colunas:**
- Datas: `format(parseISO(created_at), 'dd/MM/yyyy', { locale: ptBR })`
- Venda: `<Link href={/vendas/${row.sale_id}} className="text-primary hover:underline font-medium">{row.sale_id.slice(0,8).toUpperCase()}</Link>` — ou abreviação visível do ID
- Monetário: `formatCurrency(value.cents / 100)` com `tabular-nums`
- Porcentagem: `(value).toLocaleString('pt-BR', { minimumFractionDigits: 2, maximumFractionDigits: 4 }) + '%'`
- Campos nulos (`broker_id` ou `agency_id` ausentes): exibir `—`

### B4 — Filtros

```
flex flex-wrap items-center gap-3
```

Três filtros em linha, mesma estrutura que `InstallmentsFilters`:

**Filtro Corretor:**
- Componente: combobox com busca (padrão `CustomerFilter` adaptado para brokers)
- Placeholder: "Corretor"
- Search: busca por nome completo via API de corretores
- Clear: badge com × ao selecionar
- Popula `broker_id` na query

**Filtro Imobiliária:**
- Componente: combobox com busca (padrão idêntico ao de Corretor)
- Placeholder: "Imobiliária"
- Search: busca por nome via API de agências
- Clear: badge com × ao selecionar
- Popula `agency_id` na query

**Filtro Período:**
- Componente: `DateRangeFilter` (já existente no projeto)
- Placeholder: "Período"
- Presets: `thisMonth` (Mês atual), `lastMonth` (Mês anterior), `thisYear` (Ano atual)
- Popula `from` e `to` na query (formato ISO 8601)

Ao limpar qualquer filtro → volta para página 1 automaticamente.

### B5 — Paginação

```
1–10 de 47 comissões     [< 1 2 3 4 5 >]
```

- `PAGE_SIZE = 10` (fixo, sem seletor de tamanho)
- Label: `"{from}–{to} de {total} comissões"`
  - `from = (page - 1) * PAGE_SIZE + 1`
  - `to = Math.min(page * PAGE_SIZE, total)`
- Botões de página numerados com `Button variant="outline"` para página atual (highlight) e `variant="ghost"` para demais
- Botões `<` e `>` para anterior/próximo — desabilitados na primeira/última página
- Padrão: `shadcn <Pagination>` com `<Button>` para números (mesmo padrão de `InstallmentsPagination`)

### B6 — Estado vazio

Dentro da tabela, quando `data.length === 0 && !isLoading`:

```tsx
<div className="flex flex-col items-center justify-center py-12 text-center">
  <p className="text-sm text-muted-foreground">Nenhuma comissão encontrada</p>
  {hasActiveFilters && (
    <Button variant="outline" size="sm" onClick={onClearFilters} className="mt-4">
      Limpar filtros
    </Button>
  )}
</div>
```

## Referências

### Design system / tokens

- **Espaçamento:** `space-y-6` no container raiz da página (mesmo da `/financeiro`)
- **Cards totais:** `grid gap-4 grid-cols-1 sm:grid-cols-3`
- **Tipografia header:** `text-3xl font-bold tracking-tight` (h1) + `text-muted-foreground mt-2` (p)
- **Card header:** `flex flex-row items-center justify-between pb-2`
- **Valor do card:** `tabular-nums text-2xl font-bold tracking-tight`
- **Tabela container:** `Card className="rounded-xl shadow-sm"` + `CardContent className="p-0"`
- **Valores monetários:** `tabular-nums` obrigatório em todas as células com valores BRL e %

### Componentes shadcn a usar

| Componente | Uso |
|-----------|-----|
| `Card`, `CardHeader`, `CardTitle`, `CardContent` | Cards de totais e container da tabela |
| `Table`, `TableHeader`, `TableBody`, `TableRow`, `TableHead`, `TableCell` | Tabela de listagem |
| `Skeleton` | Estado de loading (cards e rows da tabela) |
| `Button` | Paginação e "Limpar filtros" |
| `Pagination`, `PaginationContent`, `PaginationItem` | Estrutura da paginação |

Componentes UI customizados a reutilizar (já existentes):
- `DateRangeFilter` — filtro de período com presets
- `BrokerFilter` (ou combobox similar a `CustomerFilter`) — filtro de corretor
- `AgencyFilter` (ou combobox similar a `CustomerFilter`) — filtro de imobiliária

### Acessibilidade

- `tabular-nums` obrigatório em todos os valores numéricos (BRL e %) para alinhamento consistente
- Link da venda: `<a href="/vendas/{id}">` com texto descritivo (não "clique aqui")

- Botão "Limpar filtros" com label explícito em português
- `TableHead` com conteúdo de texto significativo para cada coluna
