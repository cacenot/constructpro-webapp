## Story

**As a** corretor usando o formulário de nova proposta ou edição,
**I want** poder selecionar bimestral (2 meses), trimestral (3 meses) e semestral (6 meses) como periodicidade de parcelas,
**so that** eu possa estruturar planos de pagamento com intervalos maiores que o mensal, alinhados ao que a API 1.0.0 já suporta.

## Acceptance Criteria

1. O seletor de periodicidade de parcelas exibe as 3 novas opções: **Bimestral**, **Trimestral** e **Semestral** com labels em português.
2. Ao criar um grupo de parcelas bimestral/trimestral/semestral, o campo "Dia de Vencimento" funciona igual ao mensal — o usuário informa o dia e a data de início é sugerida automaticamente.
3. Datas permitidas no date picker de "Data de Início" para as novas periodicidades seguem o mesmo padrão do mensal: qualquer ocorrência do dia informado em qualquer mês (dentro dos próximos 30 anos).
4. A data de fim de contrato calculada localmente (usada no resumo/preview) respeita o intervalo correto:
   - Bimestral: `start_date + (quantity - 1) × 2 meses`
   - Trimestral: `start_date + (quantity - 1) × 3 meses`
   - Semestral: `start_date + (quantity - 1) × 6 meses`
5. Clamp de fim de mês funciona: ao calcular próximas datas, se o `recurrence_day` não existe no mês alvo (ex: dia 31 em fevereiro), usar o último dia válido daquele mês.
6. As mesmas opções estão disponíveis tanto no formulário de **criação** (`sale-form-step2.tsx`) quanto no de **edição** (`sale-edit-form.tsx`).
7. Parcelas com `recurrence_type ∈ {bimonthly, quarterly, semestral}` passam na validação Zod do `installmentScheduleSchema` sem erro.
8. Testes unitários em `tests/unit/lib/installment-utils.test.ts` cobrem os novos cálculos com **pelo menos 3 casos de borda por periodicidade** (incluindo clamp de fevereiro, meses com 30 dias, e virada de ano).

### `computeContractEndDate` — Lógica do Clamp

[Source: `src/lib/installment-utils.ts`]

**Problema atual:** `new Date(start).setMonth(start.getMonth() + N)` causa overflow no JS quando o dia não existe no mês alvo (ex: Jan 31 + 1 mês → Mar 3, não Fev 28). Isso deve ser corrigido com um helper.

**Helper a implementar:**

```ts
function addMonthsClamped(start: Date, monthsToAdd: number, dayOfMonth: number): Date {
  const baseYear = start.getFullYear()
  const baseMonth = start.getMonth() // 0-indexed
  const targetMonthTotal = baseMonth + monthsToAdd
  const targetYear = baseYear + Math.floor(targetMonthTotal / 12)
  const targetMonthIndex = ((targetMonthTotal % 12) + 12) % 12 // 0-indexed
  const daysInTarget = new Date(targetYear, targetMonthIndex + 1, 0).getDate()
  const targetDay = Math.min(dayOfMonth, daysInTarget)
  return new Date(targetYear, targetMonthIndex, targetDay)
}
```

**Uso em `computeContractEndDate`:**

```ts
// Para monthly:
const end = addMonthsClamped(start, (quantity - 1), recurrence_day)

// Para bimonthly:
const end = addMonthsClamped(start, (quantity - 1) * 2, recurrence_day)

// Para quarterly:
const end = addMonthsClamped(start, (quantity - 1) * 3, recurrence_day)

// Para semestral:
const end = addMonthsClamped(start, (quantity - 1) * 6, recurrence_day)
```

**Atenção:** Aplicar `addMonthsClamped` também ao `monthly` existente — corrige potencial bug latente.
