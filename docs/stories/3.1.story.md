## História

**Como** usuário autenticado (vendedor ou gerente),
**Quero** selecionar corretor e/ou imobiliária com suas taxas de comissão ao criar uma nova proposta,
**Para que** a intenção de comissão fique registrada desde a criação, espelhando as regras de negócio da API.

## Critérios de Aceitação

1. **Seção "Comissão"** exibida no formulário de Nova Proposta (`sale-form.tsx`), após o card "Dados da Venda", com todos os campos opcionais.
2. **Select de Corretor:** lista paginada via `GET /api/v1/brokers` (page_size=100); exibe `full_name` como label, envia `id` como valor; campo opcional.
3. **Select de Imobiliária:** lista via `GET /api/v1/agencies` (page_size=100); exibe `trade_name ?? legal_name` como label, envia `id` como valor; **habilitado somente quando corretor estiver selecionado** (desabilitado + placeholder "Selecione um corretor primeiro" caso contrário).
4. **Input Taxa Corretor (%):** visível somente quando corretor selecionado; número decimal (ex.: `1.5` para 1,5%); convertido para PPM (`Math.round(valor * 10000)`) ao submeter; obrigatório quando corretor selecionado.
5. **Input Taxa Imobiliária (%):** visível somente quando imobiliária selecionada; mesmo padrão de conversão PPM; obrigatório quando imobiliária selecionada.
6. **Validação Zod — Regra C1:** se `broker_id` estiver preenchido, `commission_broker_rate` deve ser > 0; se `agency_id` estiver preenchido, `broker_id` deve estar preenchido e `commission_agency_rate` deve ser > 0. Implementado via `.superRefine()` no `saleFormSchema`.
7. **Limpeza em cascata:** ao desmarcar corretor, `agency_id`, `commission_broker_rate` e `commission_agency_rate` são resetados para `null`/`undefined`.
8. **Submit:** os quatro campos (`broker_id`, `commission_broker_rate`, `agency_id`, `commission_agency_rate`) incluídos no body do `POST /api/v1/sales`; quando não preenchidos, omitidos ou enviados como `null`.
9. **Error handling:** `throwApiError` no `mutationFn`, `handleApiError` no `catch` — padrão Pattern B do `src/lib/api-error.ts`.
10. **Lint e build:** `npm run lint` e `npm run build` sem erros ao término.

const indexTypesQuery = useQuery({

### Endpoints da API (via `useApiClient`)

```
GET /api/v1/brokers?page=1&page_size=100
  Response: { items: BrokerResponse[], total: number, ... }
  BrokerResponse: { id: number, full_name: string, cpf: string, creci: string, ... }

GET /api/v1/agencies?page=1&page_size=100
  Response: { items: AgencyResponse[], total: number, ... }
  AgencyResponse: { id: number, legal_name: string, trade_name?: string | null, cnpj: string, ... }
```

### Regras de negócio da API (Contrato `POST /api/v1/sales`)

Documentadas no schema do api-client v1.1.0:
- **C1**: `broker_id` e `commission_broker_rate` devem ser paired (ambos ou nenhum)
- **Agency requer broker**: `agency_id` só é válido se `broker_id` também estiver presente
- **PPM**: rates são integers em parts-per-million (1% = 10.000). Conversão: `Math.round(percentual * 10000)`
- **Cap**: `commission_broker_rate + commission_agency_rate ≤ TenantConfig.max_commission_rate` (validado pela API — tratar 422 normalmente via `handleApiError`)
- Campos completamente opcionais — proposta sem comissão continua válida

### Conversão % → PPM

A UI exibe percentual (ex.: `1.5` = 1,5%). Antes do submit, converter:
```ts
Math.round(percentualValue * 10000)
// 1.5% → 15000 PPM
// 2%   → 20000 PPM
```
