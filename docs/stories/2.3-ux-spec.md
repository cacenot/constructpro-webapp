# UX Spec — Story 2.3: Módulo Imobiliária (CRUD completo)

---

## Objetivo da tela / componente

O módulo Imobiliária provê quatro telas de CRUD para gerenciamento de agências parceiras:

| Rota | Objetivo |
|------|---------|
| `/imobiliarias` | Listar, filtrar e paginar imobiliárias cadastradas. Ponto de entrada do módulo. |
| `/imobiliarias/novo` | Formulário de criação de nova imobiliária. |
| `/imobiliarias/:id` | Visualização plana de todos os dados de uma imobiliária. |
| `/imobiliarias/:id/editar` | Formulário de edição de imobiliária existente (CNPJ desabilitado). |

**Usuários-alvo:** vendedores e gerentes autenticados.
**Valor entregue:** permite manter o cadastro de agências parceiras para posterior vinculação a propostas de venda (Epic 3).

---

## Fluxo principal do usuário

### Fluxo 1 — Listar e navegar

```
Sidebar "Imobiliárias"
  → /imobiliarias (lista paginada)
    → busca por razão social / CNPJ / CRECI-J
    → paginação (mínimo 10 por página)
    → clique em ação "Ver detalhes" → /imobiliarias/:id
    → clique em ação "Editar"       → /imobiliarias/:id/editar
    → clique em ação "Excluir"      → AlertDialog de confirmação → redirect /imobiliarias
    → botão "Nova imobiliária"      → /imobiliarias/novo
```

### Fluxo 2 — Criar imobiliária

```
/imobiliarias/novo
  → preencher formulário
  → submit → toast sucesso "Imobiliária cadastrada com sucesso!"
  → redirect automático para /imobiliarias
  → erro API → toast erro (via handleApiError)
```

### Fluxo 3 — Editar imobiliária

```
/imobiliarias/:id (detalhe) → botão "Editar"
  → /imobiliarias/:id/editar (CNPJ desabilitado)
  → submit → toast sucesso "Imobiliária atualizada com sucesso!"
  → redirect para /imobiliarias/:id
  → erro API → toast erro
```

### Fluxo 4 — Soft-delete

```
/imobiliarias ou /imobiliarias/:id → botão "Excluir"
  → AlertDialog: "Excluir imobiliária?" + nome da razão social
  → confirmar → toast sucesso "Imobiliária removida com sucesso!"
  → redirect para /imobiliarias
  → cancelar → fechar dialog, sem ação
```

## Estados da UI (loading, empty, error, success)

### Página Lista `/imobiliarias`

| Estado | Comportamento visual |
|--------|---------------------|
| **Loading inicial** | Skeleton rows (10 linhas) dentro do `<Card>` — mesmo padrão de broker-table |
| **Loading ao mudar filtro/página** | Skeleton rows substituem dados atuais enquanto nova query resolve |
| **Empty sem filtro** | Ícone `Landmark` centralizado + "Nenhuma imobiliária cadastrada" + botão "Cadastrar imobiliária" |
| **Empty com filtro ativo** | Ícone `Search` centralizado + "Nenhuma imobiliária encontrada para esta busca" + botão "Limpar filtros" |
| **Dados carregados** | Tabela com linhas, paginação e info "X–Y de Z imobiliárias" |
| **Erro de rede** | Manter skeleton ou mostrar estado vazio com mensagem "Erro ao carregar imobiliárias" |

### Páginas Formulário (criação / edição)

| Estado | Comportamento visual |
|--------|---------------------|
| **Loading (edição — carregando dados)** | Skeletons nos campos do Card enquanto `useQuery` resolve |
| **Formulário pronto** | Campos editáveis, CNPJ desabilitado em edição (`disabled` + opacidade reduzida) |
| **Submitting** | Botão principal: `<Loader2 animate-spin /> Salvando...`, todos campos desabilitados |
| **Erro de validação** | `<FormMessage>` abaixo do campo com mensagem em português |
| **Sucesso** | Toast `bottom-center` com `richColors`: "Imobiliária cadastrada/atualizada com sucesso!" + redirect |
| **Erro API** | Toast erro via `handleApiError` |

### Página Detalhe `/imobiliarias/:id`

| Estado | Comportamento visual |
|--------|---------------------|
| **Loading** | Skeleton: botão voltar, título, card com 4 linhas de skeleton |
| **Dados carregados** | Layout flat (sem tabs) com card "Dados da Imobiliária" |
| **Not found / erro** | Mensagem "Imobiliária não encontrada" + botão "Voltar para lista" centralizado |

## Comportamentos esperados (interações, validações visuais)

### Sidebar

- **Ícone recomendado:** `Landmark` (lucide-react) — diferencia visualmente de `Building2` já usado em "Empreendimentos" (navOperacoes).
- Entrada adicionada ao grupo `navComercial` após "Corretores":
  ```ts
  { title: 'Imobiliárias', href: '/imobiliarias', icon: Landmark }
  ```
- Link ativo destacado automaticamente via `isActive()` existente.

### Tabela de Lista

**Colunas:**

| Coluna | Header | Visibilidade | Notas |
|--------|--------|-------------|-------|
| Razão Social + CNPJ | "Razão Social" | Sempre | Nome em `font-medium`, CNPJ mascarado em `text-xs text-muted-foreground tabular-nums` abaixo |
| Nome Fantasia | "Nome Fantasia" | `hidden md:table-cell` | Exibir `—` (en dash) quando null/ausente |
| CRECI-J | "CRECI-J" | Sempre | `text-sm text-muted-foreground` |
| E-mail | "E-mail" | `hidden md:table-cell` | `—` quando ausente |
| Ações | "" | Sempre | Dropdown com Tooltip |

**Dropdown de ações (padrão canônico):**
```
DropdownMenuLabel: "Ações"
DropdownMenuItem: "Ver detalhes"    → /imobiliarias/:id
DropdownMenuItem: "Editar"          → /imobiliarias/:id/editar
DropdownMenuSeparator
DropdownMenuItem: "Excluir"         → abre AlertDialog (text-destructive)
```
- Trigger do dropdown envolto em `<Tooltip>` com label "Ações".
- Itens sem ícone (padrão CLAUDE.md).

**Busca:**
- Placeholder: `"Buscar por razão social, CNPJ ou CRECI-J..."`
- Debounce 300ms, reset para página 1 ao alterar.
- Botão "Limpar filtros" aparece apenas quando `hasActiveFilters === true`.

**Paginação:**
- Info: `"X–Y de Z imobiliárias"`
- Page size mínimo: 10 itens.

### Formulário (criação e edição)

**Layout do Card — "Dados da Imobiliária":**

```
Row 1: [CNPJ — col-span-4/12] (disabled em edição)
Row 2: [Razão Social — full width]
Row 3: [Nome Fantasia — full width] (opcional)
Row 4: [CRECI-J — full width]
Row 5: [Telefone — col 1/2] [E-mail — col 2/2]
```

**Campos:**

| Campo | Label | Obrigatório | Componente | Comportamento especial |
|-------|-------|-------------|-----------|------------------------|
| `cnpj` | "CNPJ *" | Sim | `<CNPJInput>` de `document-input.tsx` | `disabled` em edição, máscara `XX.XXX.XXX/XXXX-XX` |
| `legal_name` | "Razão Social *" | Sim | `<Input>` | `maxLength={160}` |
| `trade_name` | "Nome Fantasia" | Não | `<Input>` | `placeholder="Nome fantasia (opcional)"`, `maxLength={120}` |
| `creci_j` | "CRECI-J *" | Sim | `<Input>` | `placeholder="Ex: CRECI-SP 12345-J"`, `maxLength={20}` |
| `phone` | "Telefone" | Não | `<PhoneInput>` | `defaultCountry="BR"`, sem prop `international` |
| `email` | "E-mail" | Não | `<Input type="email">` | `placeholder="email@exemplo.com"`, `maxLength={255}` |

**Labels obrigatórios:** campos obrigatórios usam label com `" *"` suffix (padrão corretor).

**Botões de ação (rodapé do formulário):**
```
[Cancelar — variant="outline"]   [Cadastrar / Atualizar — variant="default" + <Save icon>]
```
- Em submitting: `[Salvando... — variant="default" disabled + <Loader2 animate-spin>]`
- Botão "Voltar" (ghost + ícone `<ArrowLeft>`) no header do formulário com `<Tooltip>Voltar</Tooltip>`.

### Página Detalhe

**Layout flat (sem tabs) — padrão corretor:**
- Header: botão Voltar (ghost + ArrowLeft) + razão social como `h1` + botões Editar e Excluir no lado direito.
- Card "Dados da Imobiliária" com grid `sm:grid-cols-2`:
  - CNPJ (mascarado, `tabular-nums`)
  - Razão Social
  - Nome Fantasia (renderizar apenas se presente — não exibir campo vazio)
  - CRECI-J
  - E-mail (renderizar apenas se presente)
  - Telefone formatado (renderizar apenas se presente)

**Tooltips no detalhe:**
- Botão Voltar: "Voltar"
- Botão Editar: "Editar"
- Botão Excluir: "Excluir"

### AlertDialog de Exclusão

```
Título: "Excluir imobiliária?"
Descrição: "Esta ação removerá a imobiliária [razão social] do sistema. Deseja continuar?"
Botões:
  [Cancelar] (disabled durante isPending)
  [Excluir — variant="destructive"] (disabled + Loader2 durante isPending)
```

### Toasts

| Ação | Mensagem |
|------|---------|
| Criar sucesso | "Imobiliária cadastrada com sucesso!" |
| Editar sucesso | "Imobiliária atualizada com sucesso!" |
| Excluir sucesso | "Imobiliária removida com sucesso!" |
| Erro genérico | Via `handleApiError` com fallback específico por operação |

- Posição: `bottom-center`, `richColors` (configuração global do Sonner já existente).

### Acessibilidade (WCAG 2.1 AA)

- Todos os botões com apenas ícone possuem `<Tooltip>` com label em português.
- Campos de formulário com `<FormLabel>` associado via react-hook-form.
- AlertDialog usa primitivos do Radix UI (foco gerenciado, trap focus automático).
- `tabular-nums` em CNPJ e telefone para leitura consistente.

## Decisões de design tomadas

| Decisão | Escolha | Justificativa |
|---------|---------|--------------|
| Ícone sidebar | `Landmark` | `Building2` já usado em "Empreendimentos" — evitar confusão visual |
| Layout detalhe | Flat (sem tabs) | Imobiliária não tem sub-entidades no escopo desta story (mesmo padrão corretor) |
| Placeholder `trade_name` | "Nome fantasia (opcional)" | Explícito sobre opcionalidade sem depender apenas do label |
| `trade_name` na tabela quando vazio | `—` (en dash) | Consistência com padrão do módulo corretor para campos opcionais |
| Coluna CNPJ na tabela | Agrupada com Razão Social (subtext) | Economiza coluna, leitura mais natural |
| Responsividade tabela | Ocultar Nome Fantasia e E-mail em mobile | Colunas menos críticas para ação rápida; CNPJ (no subtext) e CRECI-J sempre visíveis |
